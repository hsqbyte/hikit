# 🔀 Git 客户端

> 内置轻量 Git GUI，类似 SourceTree / GitKraken 但集成在 HiKit 里

## 技术方案
- Go 后端直接调用系统 `git` 命令（不引入 git 库，保持轻量）
- 前端可视化展示 diff、分支、提交历史
- 资产树新增 `git_repo` 类型，或侧边栏独立入口

---

## ✅ Phase 1 - 仓库浏览 & 状态

- [x] **打开仓库** — 选择本地文件夹打开 Git 仓库（系统对话框）
- [x] **仓库状态** — 显示当前分支、ahead/behind 计数
- [x] **文件状态列表** — 分类展示：已暂存 / 已修改 / 未跟踪 文件
- [x] **Diff 查看** — 点击文件弹出 diff 对比（绿色新增/红色删除）
- [x] **暂存 / 取消暂存** — 点击 + / - 按钮，支持单文件或全部
- [x] **提交** — 输入 commit message 提交（Cmd+Enter 快捷键）
- [x] **丢弃更改** — 单文件丢弃

## ✅ Phase 2 - 分支 & 历史

- [x] **分支列表** — 展示本地/远程分支，当前分支高亮
- [x] **切换分支** — 点击分支名切换（checkout）
- [x] **新建分支** — 从当前 HEAD 创建分支
- [x] **提交历史** — 时间线展示 commit log（哈希、作者、消息、时间）
- [ ] **删除分支** — 删除本地分支（带保护确认）
- [ ] **合并分支** — 选择目标分支合并到当前分支
- [ ] **Commit 详情** — 点击 commit 查看改动文件列表和 diff

## ✅ Phase 3 - 远程操作

- [x] **Push** — 推送到远程仓库
- [x] **Pull** — 拉取远程更新
- [x] **Fetch** — 获取远程信息
- [ ] **远程管理** — 查看/添加/删除 remote

## Phase 4 - 高级功能

- [ ] **Stash 管理** — 暂存工作区 / 恢复 / 删除 stash
- [ ] **冲突解决** — 合并冲突可视化标记 + 选择保留版本
- [ ] **标签管理** — 创建/删除 Tag
- [ ] **Commit Graph** — 分支合并可视化图谱（类似 git log --graph）
- [ ] **.gitignore 编辑** — 快捷编辑忽略规则
- [ ] **Blame 视图** — 逐行显示最后修改者
